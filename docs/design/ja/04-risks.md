
# 4. リスクと緩和策

## 4.1 技術リスク

| リスク | 重大度 | 緩和策 |
| --- | --- | --- |
| Web互換性が実用ブラウザとして低すぎる | 高 | Compat層（Ch. 7）がエスケープハッチ。クローラーデータで優先順位付け。基準は「WPT全合格」ではなく「モダンサイトで十分動作」。 |

> **Phase 0 サーベイ結果（第29章 §29.5）:** 具体的な compat 優先度が確定 — P0: width/height 60%+、-webkit- ~40% EN; P1: appearance、font-smoothing; P2: 非推奨タグ < 5%。
| 自作JSエンジンの成熟度 | 高 | 段階的アプローチ（パーサー → インタプリタ → IC → JIT）。Phase 1–3を通じてSpiderMonkeyがフォールバック。elidex-appは軽量ワークロードのためelidex-jsを先行採用可能。ES2020+のみのCoreがAnnex Bを省略し大幅に簡素化。 |
| 実サイト向けDOM API完全性 | 中 | モダンフレームワーク（React、Vue、Svelte）は生JSよりDOM APIサーフェスが小さい。レガシーAPIはクローラーの破損データに基づきオンデマンド実装。 |

> **Phase 0 サーベイ結果（第29章 §29.4）:** document.all 0%（非優先化を検証）、document.write JA 12.4% / EN 5.3%（広告/アナリティクス主体 — strict-only アプローチを支持）。
| Velloのプロダクションブラウザ利用成熟度 | 中 | VelloはLinebenderプロジェクトで活発に開発中。接触面は分離済み（Ch. 15 §15.6、ADR #26）— 上流コードはVello依存ゼロ。ソフトウェアフォールバック（Vello CPUバックエンド）でレンダリングは常に動作。 |
| プラグインシステムのパフォーマンスオーバーヘッド | 中 | ホットパスではdynamic dispatch（traitオブジェクト）ではなくstatic dispatch（ジェネリクス/enum）。プラグインはランタイムではなくビルド時に解決。 |
| ECSメモリモデルとDOMセマンティクスの不一致 | 中 | すべての関係が1:1エンティティ-コンポーネントではない（例：LayerTreeは独立、ADR #27）。ハイブリッドアプローチ：DOMにはECS + N:M関係には独立構造。 |
| コーデック特許・ライセンスリスク | 中 | 特許負担コーデック（H.264、H.265、AAC）はプラットフォームデコーダーにのみ委譲（ADR #34）。elidex配布物に特許負担コードなし。 |

## 4.2 スコープ＆リソースリスク

| リスク | 重大度 | 緩和策 |
| --- | --- | --- |
| Webプラットフォーム複雑性によるスコープクリープ | 高 | プラグインアーキテクチャが各機能を離散的・除去可能な単位に強制。厳格なフェーズゲートで基盤が堅固になる前の機能追加を防止。 |
| 単一開発者ボトルネック | 高 | モジュラークレート構造でコントリビューター参加時の並行開発を可能に。独自価値（コアエンジン）に集中し、既存クレート（rustybuzz、cssparser、html5ever、mozjs）を活用。 |
| WebRTCの複雑性 | 中 | WebRTC完全実装は延期（Ch. 20 §20.11）。将来統合用のインターフェースを定義。準備時にwebrtc-rs等の既存Rust WebRTCライブラリを使用可能。 |
| DRM/Widevineライセンス | 中 | EME APIをCDMプラグインスロット付きで定義（Ch. 20 §20.8）。v1はCDMなしで出荷。アーキテクチャ的準備によりライセンス取得時の再設計不要。 |

## 4.3 エコシステムリスク

| リスク | 重大度 | 緩和策 |
| --- | --- | --- |
| 重要Rustクレートのメンテナンス停止 | 中 | 重要依存関係（rustybuzz、wgpu、Vello、AccessKit）は確立チームが活発にメンテナンス。Trait抽象化パターン（ADR #31、#34）で実装の差し替えを可能に。 |
| プラットフォームAPI変更 | 低 | Platform Abstraction層（Ch. 23）がプラットフォーム固有を分離。変更はアダプターのみに影響。 |
| ブラウザエンジン集約圧力 | 中 | elidexのデュアルユースモデル（ブラウザ + アプリランタイム）がブラウザ市場シェアが小さくても価値を提供。elidex-appはChromeではなくElectron/Tauriと競合。 |
