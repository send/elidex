## §1 Problem

`SetTextContentNodeKind::invoke` (`crates/dom/elidex-dom-api/src/node_methods/text_content.rs:105-116`) Element/DocumentFragment branch:
```rust
let children = dom.children(this);
for child in children { session.release(child); let _ = dom.remove_child(this, child); }  // raw EcsDom, NO record
if !text.is_empty() { let text_node = dom.create_text(text); let _ = dom.append_child(this, text_node); }  // raw, NO record
```
1. **MO-silent.** Raw `EcsDom::remove_child`/`append_child` produce **no** `MutationRecord`; `MutationObserver({childList:true})` never observes a textContent replace. The last childList-mutating surface in B1.2 still bypassing the record path (every other tree op converged in #379/#384/#387/#393/#399/#402).
2. **One-issue-one-way violation.** The replace-all algorithm has a canonical record-producing home (`apply_replace_all`); this branch is a second, MO-silent copy of it.
3. **Sub-fidelities the canonical layer already gets right** (this branch does not): `child_list_uncapped` removedNodes (no MAX_ANCESTOR_DEPTH truncation, #387 discipline); shadow-root reject; the §4.2.3-step-7 "queue iff added∪removed non-empty" empty-no-op; `node document` of the fresh Text.

## §2 Coupled invariants at this corner (the design surface / edge matrix)

1. **§4.4 setter node construction** (WHATWG DOM §4.4 `Node.textContent` setter, Element/DocumentFragment): "Let node be null. If the given value is non-empty, set node to a new `Text` whose data is the value and **node document is this's node document**. Replace all with node within this." → `node = text.is_empty() ? None : Some(create_text_with_owner(text, owner_document(this)))`. (Today's `create_text` omits the owner document — a fidelity gain.)
2. **Record shape & count** (via `apply_replace_all`): exactly **ONE** coalesced `ChildList` record on `this` — `removedNodes` = prior children (uncapped), `addedNodes` = `[text]` (non-empty) or `«»` (empty string), prev/next null. Empty string on an already-empty element = added∪removed empty → **no record** (§4.2.3 step 7). A fresh Text is neither a fragment nor already-parented, so no source-removal/fragment sub-record — strictly 1 record (or 0).
3. **`session.release` drop** (One-issue-one-way + MO correctness): the prior per-child `session.release` is removed to match the `replaceChildren` convergence (`child_node/mutations.rs:351` calls `apply_replace_all` with no release). Releasing a removed child's wrapper would break SameObject for a removed node still referenced by script + contradict that the node appears in `removedNodes`; the wrapper is GC-reclaimed normally. No behavior depends on the eager release (it was a pre-record-path cleanup).
4. **NodeKind dispatch unchanged** (the `match dom.node_kind(this)` arms): Document/DocumentType → no-op (return Undefined); Text/CdataSection → `set_text_data` (characterData, B1.3, untouched); Comment → `CommentData` write (characterData, B1.3, untouched); `_` (Element/DocumentFragment/etc.) → the replace-all branch THIS slice converges. No arm added/removed.
5. **Dual-runtime / leak-guard** (identical to #393/#399/#402): the handler `push_notify_record`s for all runtimes; the VM drains at `invoke_dom_api` Phase 2.5, boa/wasm discard via `SessionCore::flush` leak-guard. The VM already routes every textContent setter (`node_proto.rs`, `html_{script,anchor,output,title,textarea,option}_proto.rs`, `html_option_proto`) through `invoke_dom_api("textContent.set", ...)` — **no VM change needed**; they all gain records by routing through the converged handler.
6. **Shadow-root receiver / children** (DOM §4.8): `apply_replace_all` rejects a shadow-root *node* arg (n/a — node is a fresh Text), and `child_list_uncapped`/`children` of a host return light-tree children only (shadow root excluded), so a textContent set on a shadow host clears its light children, never the shadow tree — preserved by reusing the canonical primitive.

## §4 Design decision: reuse `apply_replace_all` (no new primitive)

The ideal (One-issue-one-way + layering-correct): the Element branch calls the canonical `apply_replace_all` and pushes its records — the same path `replaceChildren` uses. No new handler, no new primitive, no VM change. Rejected alternative — "teach the inline branch to also push records": perpetuates the second replace-all copy (the very strangler One-issue-one-way forbids) and re-derives the uncapped/shadow/empty-no-op fidelities by hand.

**Crate-API / layering mapping:** algorithm home unchanged = `apply_replace_all` (`elidex-script-session/src/mutation/mod.rs`); the change is one branch of the engine-independent handler `SetTextContentNodeKind` (`elidex-dom-api/src/node_methods/text_content.rs`); VM (`vm/host/*`) untouched (already dispatches). `create_text_with_owner` + `owner_document` (`elidex-ecs`).

## §5 Implementation

### 5.1 `SetTextContentNodeKind` Element/DocumentFragment branch (`node_methods/text_content.rs`)
Replace the `_` arm body:
```rust
_ => {
    // WHATWG DOM §4.4 setter: node = null (empty) or a fresh Text whose node
    // document is this's; "replace all with node" = §4.2.3 replace-all → one
    // coalesced ChildList record (removed = old children, added = [text] | «»).
    let node = if text.is_empty() {
        None
    } else {
        let owner = dom.owner_document(this);
        Some(dom.create_text_with_owner(text, owner))
    };
    for record in elidex_script_session::apply_replace_all(dom, this, node) {
        session.push_notify_record(record);
    }
    Ok(JsValue::Undefined)
}
```
Drop the `children` remove-loop + per-child `session.release` + the manual `create_text`/`append_child`. Import `apply_replace_all`.

## §8 Risk / review-tail pre-emption
- **`session.release` drop** (highest-uncertainty): `SessionCore::release` = `cssom_sheets.remove(entity)` + `identity.release_entity(entity)` (wrapper-identity clear). The `replaceChildren` convergence (`apply_replace_all`, same removed-children set) does **neither** — so today textContent and replaceChildren already **diverge**, and aligning textContent to the canonical (no-release) path removes that divergence. The wrapper clear must NOT happen (removedNodes must stay valid JS objects for MO + SameObject). The only load-bearing facet is `cssom_sheets.remove` for a removed nested `<style>`/`<link>`: if that cleanup is genuinely needed it belongs at the removal **chokepoint** (`apply_remove_child` / despawn), applied uniformly to every remove path (replaceChildren, removeChild, …), NOT special-cased in textContent — folding it here would re-introduce the per-caller strangler. Plan-review to rule whether the cssom_sheets cleanup is a real gap (→ separate chokepoint slice) or dead (entity-keyed, reclaimed on despawn).
- **owner-document change** (`create_text`→`create_text_with_owner`): fidelity gain (spec "node document is this's"); for a detached `this`, `owner_document` returns the tree-root/None fallback (same as today's `create_text` which set no owner) — assert ownerDocument in a connected case.
- **Empty-no-op**: `el.textContent=''` on empty element must NOT emit a record (some naive impls emit an empty record) — `apply_replace_all` step-7 guard handles it; regression-test it.
- Edge-density is low (reuses a landed canonical algorithm; small NodeKind matrix) but the umbrella mandates plan-review; high enough blast (both runtimes + a hot DOM API) → `/external-converge` after the pre-push 6-stage gate.
