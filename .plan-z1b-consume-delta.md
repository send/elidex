# Z-1b consume — plan DELTA v2 (Option D: run-start N-fragment InlineFlow)

Parent: [[terminal-z-z1b-ifc-fold-plan]] (§7 defers the consume steps to this delta). **v2 reframe**:
plan-review pass-1 (5-agent, this session) surfaced **F1/F2/F3** — all rooted in the parent's choice
to key the per-column **text** on the **container** (→ render consumes the standalone store at the
**entity-walk** level → must paint the container per-column → **subtree-skip** → drops positioned

in-column = 1-fragment InlineFlow) and I-paged (#282, N-fragment by page generation).

**Net of Option D vs the parent's store-consume:** Z-1b = **multicol mid-break TEXT per-column
correctness**, **strictly no-regression**, **ZERO render change** (the converged `emit_inline_flow`
already paints all fragments on screen). The standalone `FragmentTree` **box** store (Z-1a dark data)
stays **dark**; the render fragment-walk + per-column **chrome** convergence + box-removal all move to
**committed-next** (where the store is consumed for ALL box types). Scope still atomic: carrier write +
N-fragment build + the legacy mid-break render route death land together.

## B. Consume-step design (Option D)

### B1. Carrier `ColumnFlowSlice` — per-group (run-start → lines), drained into `ColumnFragment`
**`ColumnFlowSlice(Vec<(Entity, Vec<InlineFlowLine>)>)`** — a transient ECS component on the IFC
**container** (`parent_entity`), carrying **per run-start group** the column's folded lines (the
existing per-group fold, `inline/mod.rs:967-980`). Written by the IFC layout for the column mid-break
case (`do_carrier = frag_is_column && !column_is_whole`); **drained** (get+remove) by multicol fill in
the box-snapshot capture loop (`fill.rs:188`, keyed on the same mid-break container = `parent_entity`);
**never read by render**. Side-store→component rule: per-entity, `Send+Sync`, not a per-VM handle →
component is correct; write-then-drain-within-one-pass = transport (the line slice is produced deep in
`stack_block_children` per column while the column offset is only known later in
`position_column_fragments`' loop var `i` — no return-value channel). `ColumnFragment.box_snapshots`
→ a struct `FragmentSnapshot { entity, box_fragment, flow_groups: Vec<(Entity, Vec<InlineFlowLine>)> }`
(box push stays Z-1a; `flow_groups` = the drained carrier, empty when no IFC carrier). Reconcile:
insert-or-remove on the container (early returns also `remove_one`); a leak is benign (drained-only,
render never reads it).

### B2. `position_column_fragments` — accumulate per-run-start lines, build ONE-fragment InlineFlow AFTER the col loop
Box push **UNCHANGED** (Z-1a dark store). NEW: accumulate `flows: HashMap<Entity, Vec<InlineFlowLine>>`
across the column loop — for column `i`, per `(run_start, lines)` in each snapshot's `flow_groups`:
clone the lines, apply the **column delta** (WM-projected inline-axis: `inline_start += i*(w+gap)`,
`block_start += 0` — same projection as the box delta), `flows[run_start].extend(offset_lines)`. After
the loop (guarded `!is_probe`): `insert_one(run_start, InlineFlow::single(layout_generation, all_lines))`
per run-start. **One fragment, generation = `layout_generation` (F2 fix, Axis-4):** per
`components.rs:226-229` multicol columns get generation **`0`** (non-paged) — they "coexist on one
surface at absolute coords", discriminated by the **baked per-column offsets**, NOT by a per-column
generation index. So all columns' lines fold into **one** `InlineFragment` (matching whole-in-column
`InlineFlow::single`, `inline/mod.rs:1019`) — `emit_inline_flow` (screen `expected=None`) paints the
fragment's full line set = all columns at their baked offsets. (A per-column-generation N-fragment shape
would contradict the doc + the whole-in-column precedent.) **Build-after-loop is the double-shift fix**:
the per-column `shift_descendants_excluding_own_fragments` runs DURING the loop and its InlineFlow arm
finds **no** InlineFlow on the run-starts yet (cleared by the IFC's `clear_inline_flows`, not rebuilt
till after) → no-op; the col shift still moves the box LayoutBox (G11 dark) + atomics. The
born-absolute InlineFlow (offsets baked at commit) is thus excluded from the col shift **by
construction** (timing), with NO new exclusion machinery. The per-column continuation geometry (each
column's lines rebased to the column block-start) comes from the slice/rebase gate extension (A).

### B3. inline/mod.rs do_carrier path — carrier write + atomic reposition, no IFC-persist
For `do_carrier` (proven mutually exclusive with `persist_flow`): extend the `slice_and_rebase_fragment`
gate; build the per-group folded lines (the existing `:967-980` fold); run the **static-atomic** (`:995`)
and **relpos-atomic** (`:1033`) reposition (atomics positioned to their on-line spot at col-0 base, then
col-shifted by the col_children `shift_descendants` in `position_column_fragments`); write
`ColumnFlowSlice(groups)` on `parent_entity` (the per-group `(run_start, lines)` pairs). The IFC does
NOT `insert_one(InlineFlow)` for the mid-break case (lines → carrier → multicol lib). Restructure the
persist block so the fold + atomic reposition are shared between the `persist_flow` arm (sink =
`insert_one(group_key, InlineFlow::single)`) and the `do_carrier` arm (sink = carrier groups). Early
returns + non-carrier path: `remove_one::<ColumnFlowSlice>(parent_entity)` reconcile.

### B4. Render — ZERO change
`emit_inline_run` (`inline.rs:104`) already routes a run-start with an `InlineFlow` to `emit_inline_flow`,
which (screen, `expected=None`) paints **all** fragments → all columns at their baked per-column
offsets, walking atomics by entity. The mid-break run-start now HAS an N-fragment InlineFlow → the
converged path fires, the **legacy single-linear mid-break render route is dead** (the Z-1b goal). No
`emit_flow_lines` split, no store-router, no subtree-skip, no `LayoutBox` removal.

## C. Why Option D is strictly no-regression (F1/F2/F3 moot)
The mid-break container is walked by render **normally** (Option D changes only that its inline run-start
now carries an N-fragment InlineFlow instead of nothing):
- **F1 (CSSOM/a11y/hit-test)**: the container's `LayoutBox` is **kept** (Z-1a G11 box, unchanged) →
  `getBoundingClientRect`/`offset*`/a11y bounds/hit-test read it exactly as today. **No regression.**
- **F2 (positioned/abspos descendants)**: render does **not** skip the subtree → Layer 6/7 paints them
  exactly as today (G11-box-relative). **No regression.**
- **F3 (transform/clip on the container)**: normal walk → `element_transform`/clip applied exactly as
  today. **No regression.**
- **per-column TEXT**: now correct (N-fragment InlineFlow per column) — the Z-1b deliverable.
- **per-column CHROME** (background spanning columns) stays G11 single-column (no improvement, **no
  regression**) → committed-next, when render walks the box store for per-fragment chrome.
- **atomic in a mid-break IFC**: `AtomicBox` runs carried in the per-column fragments; the atomic
  `LayoutBox` is col_children-shifted (multi-column-split atomic = pre-existing G11, committed-next, no
  regression). Note: mid-break atomics are now repositioned to their on-line spot (previously the
  do_carrier case did not persist → no reposition); this is an **intended improvement**, recorded as
  churn.

## E. Edge matrix (parent §8 + Option-D-specific)
- **column-break shape**: continuation (`skip_lines>0`) / truncation (`break_after_line`) / both (3+
  columns, the middle column's slice = one fragment) / IFC starting fresh in a non-first column.
- **N-fragment InlineFlow screen paint**: all columns paint (expected=None); assert each column's lines
  at the correct per-column inline offset + column block-start (continuation rebase).
- **double-shift guard (B2 build-after-loop)**: assert the run-start InlineFlow is NOT additionally
  col-shifted (its fragments hold the baked offset exactly once) — the highest-attention Option-D edge.
- **dual-home**: whole-in-column (1-fragment InlineFlow via IFC persist + `shift_descendants`) vs
  mid-break (N-fragment via multicol-lib build); a mid-break run-start adjacent to a whole-in-column
  sibling — assert each consumed once, no double-paint.
- **box-suppression downstream (repeat trap)**: a suppressed collapsible-only line absent from the
  per-column `InlineFragment.lines` (the packer slice honors `contributes_content`); + `line_count` /
  `first_baseline` / `entity_bounds` / static_positions consistent.
- **atomic in mid-break IFC**: static/relpos/sticky atomic — `AtomicBox` in the right column fragment,
  reposition + col shift, no double-offset.
- **vertical WM multicol**: column offset = block-axis; the line column-delta projection (`inline_start`
  along the block axis) WM-correct (reuse the box delta projection).
- **spanner / non-fragmented / nested-block mid-break (D-Z2)**: no carrier / no InlineFlow build (spanner
  = `fragmentainer None`; nested-block mid-break stays legacy/G11, asserted no carrier).
- **balanced-fill probes (P1)**: probe passes write+drain into discarded ColumnFragments + `is_probe`
  suppresses the InlineFlow build (same guard as the box push) → exactly one InlineFlow per run-start.
- **reconcile across passes**: a run-start mid-break last pass, whole/non-fragmented this pass → IFC
  `clear_inline_flows` clears it, the normal persist rebuilds (1-fragment) → no stale N-fragment.

Downstream consumers to re-verify (parent §8): `InlineFlow.fragments` (now N for mid-break), the col
shift InlineFlow arm (no-op timing), atomic walk, bidi reorder (per fragment), the multicol goldens.
